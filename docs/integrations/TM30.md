# ทร.30 (TM30) Integration

ตามกฎหมายไทย โรงแรม/ที่พัก ต้อง**แจ้งการพักของชาวต่างชาติ**ภายใน 24 ชั่วโมง — มิฉะนั้นปรับสูงสุด ฿2,000 ต่อคน

## 📊 ใครต้องแจ้ง

✅ **ต้องแจ้ง**:
- ชาวต่างชาติทุกประเภทวีซ่า (ท่องเที่ยว, ธุรกิจ, นักเรียน)
- ผู้ที่ไม่มีสัญชาติไทย แม้ถือ permanent residence

❌ **ไม่ต้องแจ้ง**:
- คนไทย
- คนต่างชาติที่อยู่ไม่เกิน 24 ชม. (transit)

---

## 🌏 สถานการณ์ปัจจุบันในแต่ละพื้นที่

### Bangkok ✅ มี Online System
[extranet.immigration.go.th](https://extranet.immigration.go.th) — submit ออนไลน์ได้

### พัทยา/ชลบุรี ✅ มี Online System  
ใช้ portal เดียวกัน

### ภูเก็ต/เชียงใหม่/หัวหิน ⚠️ Mixed
- บางสำนักงานรับ online
- บางสำนักงานยังต้อง walk-in หรือ fax
- ต้องเช็คกับ ตม. ในพื้นที่

### จังหวัดอื่นๆ 📄 Manual ส่วนใหญ่
- ส่ง Form ทร.30 เข้าตม. ทุกวัน
- หรือใช้ฟอร์ม PDF ส่งทางอีเมล

## ขั้นตอน Setup

### 1. ลงทะเบียน "สถานประกอบการที่พัก" กับ ตม.

#### Bangkok / พัทยา (Online):
1. เตรียมเอกสาร:
   - หนังสือรับรองนิติบุคคล (ไม่เกิน 3 เดือน)
   - ใบอนุญาตประกอบกิจการโรงแรม (ถ้ามี)
   - แผนผังโรงแรม
   - บัตรประชาชนกรรมการ
2. ไปที่สำนักงานตรวจคนเข้าเมืองในเขต (เช่น แจ้งวัฒนะ, สวนพลู)
3. กรอก แบบ ทท.5 (ทะเบียนสถานประกอบการที่พัก)
4. รอประมาณ 7-14 วัน — รับ **Username** + **Password** สำหรับ extranet

#### ต่างจังหวัด:
1. ติดต่อ ตม. จังหวัด
2. แต่ละที่กระบวนการต่างกันเล็กน้อย — ถามทาง ตม. ตรงๆ
3. มักได้ฟอร์ม + วิธีส่งทาง email/fax

### 2. ทำความเข้าใจ Form ทร.30

ฟอร์ม ทร.30 ต้องมี:

| Field | ตัวอย่าง |
|---|---|
| ชื่อสถานประกอบการ | Maitri Hotel |
| เลขที่ทะเบียน | (จาก ตม.) |
| ชื่อ-นามสกุลแขก | John Smith |
| สัญชาติ | American |
| เลขหนังสือเดินทาง | A12345678 |
| วันเดือนปีเกิด | 1985-05-15 |
| วันที่เข้าพัก | 2026-05-10 |
| วันที่คาดว่าจะออก | 2026-05-15 |
| ห้องพัก | 305 |

### 3. Configure Maitri

```bash
# .env.local
TM30_PROVIDER=immigration
TM30_USERNAME=your_extranet_username
TM30_PASSWORD=your_extranet_password
TM30_HOTEL_REGISTRATION_NUMBER=BKK-XXXX-2026
```

> **Note**: API ของกรมตรวจคนเข้าเมือง**ไม่เปิดสาธารณะ** — Maitri ใช้แบบ:
> 1. **Auto-fill form** + export CSV → admin upload manual
> 2. **Email submission** → ส่ง email พร้อม attachment
> 3. **(Future) Direct API** หากกรมเปิด

### 4. Workflow ใน Maitri

```
1. แขกชาวต่างชาติ check-in → ระบบเช็ค nationality !== 'Thai'
   └─ Auto-create row ใน tm30_reports table
   └─ status: 'pending'

2. End of day (หรือ real-time):
   ├─ Option A: Auto-export CSV → Upload to extranet
   ├─ Option B: Auto-email to Immigration office
   └─ Option C: Direct API (เมื่อพร้อม)

3. Mark status: 'submitted' พร้อม confirmation number
```

### 5. Manual Upload Workflow

ถ้ายังไม่มี API direct, Maitri export ให้:

#### Daily Export
1. Maitri dashboard → **บัญชี → ทร.30 รอแจ้ง**
2. คลิก **Export ทั้งหมด** → ได้ CSV หรือ Excel
3. Login เข้า extranet.immigration.go.th
4. Upload file → ระบบ ตม. import
5. ได้ confirmation number → กรอกกลับใน Maitri

#### Per-Guest Export (ฉุกเฉิน)
1. หน้า reservation → คลิก **Export ทร.30 (PDF)**
2. ปริ้นต์หรือส่ง fax เข้า ตม. โดยตรง

## 🔧 Customization

### เพิ่มสำนักงาน ตม.​ ใหม่
ถ้าโรงแรมอยู่ในจังหวัดที่ยังไม่ support:

1. ไปที่ `src/lib/compliance/index.ts`
2. เพิ่ม config:
```typescript
const IMMIGRATION_OFFICES = {
  'BKK': {
    email: 'tm30@immigration.go.th',
    portal: 'https://extranet.immigration.go.th',
    method: 'extranet',
  },
  'CHIANG_MAI': {
    email: 'cm.immigration@gmail.com',
    method: 'email',
  },
  // ... เพิ่มที่นี่
};
```

### Auto-detect Foreign vs Thai
ระบบเราตรวจจาก `guests.nationality`:
- "Thai" / "TH" / "ไทย" → skip
- อื่นๆ → trigger TM30

ปรับ logic ใน `src/lib/compliance/index.ts` → `tm30Service.shouldReport()`

## 📊 Tracking & Audit

### View All Reports
```sql
-- ทร.30 ที่ยังไม่ส่ง
SELECT * FROM tm30_reports
WHERE status = 'pending'
  AND created_at < NOW() - INTERVAL '20 hours'
ORDER BY created_at;
```

### Export Monthly Report
Maitri dashboard → **Reports → TM30 Monthly Summary**:
- จำนวนแขกต่างชาติทั้งหมด
- จำแนกตามสัญชาติ
- จำนวนรายงานที่ส่งทันเวลา / สาย / ตกหล่น

ใช้สำหรับ:
- Audit จาก ตม.
- รายงานต่อผู้บริหาร
- Compliance evidence

## 🐛 Troubleshooting

### "ตม. บอกว่าไม่ได้รับรายงาน"
1. ตรวจสอบ `tm30_reports.status` = 'submitted'
2. ตรวจสอบ confirmation number ตรงกับใน extranet
3. ตรวจสอบส่งภายใน 24 ชม.
4. Provide audit trail ให้ ตม. (PDF report)

### "Portal extranet down"
- ตม. portal มี downtime บางครั้ง (ตี 2-4)
- Maitri จะ retry 3 ครั้ง อัตโนมัติ
- ถ้ายังไม่ได้: email submission เป็น fallback

### "Passport not detected"
ระบบ scan passport (Phase 2 feature) — ตอนนี้ต้อง type manual:
- หน้า reservation → กรอก passport_number, nationality, dob
- Maitri จะใช้ข้อมูลนี้สำหรับ TM30

## 📋 Compliance Best Practices

1. **เก็บข้อมูล 5 ปี**: กฎหมายกำหนด — ใช้ Supabase backup
2. **Privacy by design**: passport ขั้นต่ำ + encryption at rest
3. **PDPA-compliant**: แจ้งแขกว่าเก็บข้อมูลเพื่อทำตามกฎหมาย ตม. — ไม่ใช้ marketing
4. **Auto-submit**: ตั้ง cron job ส่งวันละ 2 ครั้ง (เที่ยงและเที่ยงคืน)
5. **Daily check**: monitor `pending` status — alert ถ้ามีค้างเกิน 20 ชม.
